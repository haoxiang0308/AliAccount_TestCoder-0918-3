# Elixir Compile-Time Information Macro

This project demonstrates an Elixir macro that prints compile-time information and saves it to a randomly named `.ex` file.

## Files

- `compile_info_macro.ex`: Defines the `CompileInfoMacro` module with the `print_and_save_compile_info/1` macro
- `test_compile_info.ex`: A test module that uses the macro
- `demo_macro.exs`: A demo script that demonstrates the macro functionality
- `compile_info_*.ex`: Randomly named files generated by the macro containing compile-time information

## How It Works

The macro `print_and_save_compile_info/1`:

1. Captures compile-time information including:
   - Module name
   - File path
   - Line number
   - Timestamp
   - Node information
   - Elixir version
   - OTP version

2. Prints the information to the console during compilation

3. Generates a random filename in the format `compile_info_[random_hex].ex`

4. Saves the compile-time information to the randomly named file

5. Creates a module attribute and function to access the compile-time information at runtime

## Usage

To use the macro in your own code:

```elixir
defmodule MyModule do
  require CompileInfoMacro
  CompileInfoMacro.print_and_save_compile_info(__MODULE__)

  # Your module functions here
end
```

The macro will automatically print compile-time information and save it to a randomly named file during compilation. The information is also stored in the module and can be accessed at runtime using `get_compile_info/0`.

## Example Output

When compiling a module that uses the macro, you'll see output like:

