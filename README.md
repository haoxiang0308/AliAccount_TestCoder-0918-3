# Rust Enum Match with Random Filename

This project demonstrates the use of `match` in Rust to handle different enum values and save the result to a file with a random name.

## Files Included:

1. **rust_enum_match_simple.rs** - The Rust implementation showing:
   - Definition of an enum with different variants
   - Using `match` to handle different enum values
   - Creating a file with a randomly generated name
   - Writing the results to that file

2. **demo_enum_match.py** - Python simulation of the same concept (used when Rust compiler wasn't available)

3. **output_*.txt** - Example output file with random name generated by the program

## Rust Enum Match Example

The Rust code demonstrates how to use `match` to handle different enum variants:

```rust
match status {
    Status::Active => "Status is Active".to_string(),
    Status::Inactive => "Status is Inactive".to_string(),
    Status::Pending => "Status is Pending".to_string(),
    Status::Error(msg) => format!("Error occurred: {}", msg),
}
```

This pattern matching is a powerful feature in Rust that allows safe and exhaustive handling of all possible enum variants.

## Random Filename Generation

The program generates a unique filename using the system timestamp to simulate randomness, then saves the results to that file.
